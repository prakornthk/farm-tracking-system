## 1. ปัญหาการเข้าใช้งาน (login, password)

### 🔐 ลืมรหัสผ่าน

**อาการ:** จำรหัสผ่านไม่ได้ ไม่สามารถเข้าสู่ระบบได้

**วิธีแก้:**

1. ที่หน้าล็อกอิน แตะ **"ลืมรหัสผ่าน?"**
2. กรอก **อีเมล** ที่ลงทะเบียนไว้
3. ระบบจะส่งอีเมลตั้งรหัสผ่านใหม่
4. เปิดอีเมลและคลิกลิงก์ตั้งรหัสผ่านใหม่
5. ตั้งรหัสผ่านใหม่ที่จำได้

**ถ้าไม่ได้รับอีเมล:**
- ตรวจสอบโฟลเดอร์ Spam/Junk
- ตรวจสอบว่าอีเมลที่กรอกถูกต้อง
- ติดต่อ super_admin เพื่อรีเซ็ตรหัสผ่านให้

### ❌ อีเมลหรือรหัสผ่านไม่ถูกต้อง

**อาการ:** ข้อความ "อีเมลหรือรหัสผ่านไม่ถูกต้อง"

**วิธีแก้:**

1. **ตรวจสอบ Caps Lock** - รหัสผ่านต้องตรงตามตัวพิมพ์ใหญ่-เล็ก
2. **ตรวจสอบอีเมล** - พิมพ์ใหม่อย่างช้า ๆ
3. **ลองกู้คืนรหัสผ่าน** - ใช้ฟังก์ชัน "ลืมรหัสผ่าน"
4. **ติดต่อผู้ดูแล** - ถ้ายังเข้าไม่ได้ ติดต่อ super_admin

### ⏳ บัญชีถูกระงับ

**อาการ:** ข้อความ "บัญชีถูกระงับ กรุณาติดต่อผู้ดูแลระบบ"

**วิธีแก้:**

1. บัญชีถูกระงับเนื่องจาก:
   - ไม่ได้ใช้งานนานเกินไป
   - ถูกระงับโดย super_admin
2. **ติดต่อ super_admin** เพื่อขอเปิดใช้งาน

### 🔒 Account Locked

**อาการ:** ข้อความ "บัญชีถูกล็อกชั่วคราว เนื่องจากใส่รหัสผ่านผิดหลายครั้ง"

**วิธีแก้:**

1. **รอ 15-30 นาที** แล้วลองใหม่
2. ใช้ฟังก์ชัน "ลืมรหัสผ่าน" เพื่อตั้งรหัสใหม่
3. ติดต่อ super_admin ถ้าต้องการปลดล็อกทันที

### 🌐 เข้าสู่ระบบไม่ได้ (Error)

**อาการ:** Error หน้าจอขาว หรือข้อความ 500, 502, 503

**วิธีแก้:**

1. **รีเฟรชหน้า** (กด F5 หรือ Ctrl+R)
2. **ลองเบราว์เซอร์อื่น** (Chrome, Firefox, Edge)
3. **ล้าง cache** ของเบราว์เซอร์
4. **ตรวจสอบ Server** - ดูสถานะ Server ว่าออนไลน์อยู่หรือไม่
5. **ติดต่อผู้ดูแล Server** ถ้าปัญหายังอยู่

## 2. ปัญหา LIFF App (สแกนไม่ได้, offline)

### 📵 สแกน QR Code ไม่ได้

**อาการ:** กล้องไม่ทำงาน หรือสแกนแล้วขึ้น Error

**วิธีแก้:**

1. **อนุญาตกล้อง** - ตรวจสอบว่าอนุญาตให้ LIFF App เข้าถึงกล้องแล้ว
   - ตั้งค่า > แอป > LINE > กล้อง > อนุญาต
2. **ตรวจสอบแสง** - QR Code ต้องมีแสงเพียงพอ
3. **ตรวจสอบ QR Code** - ดูว่า QR Code ไม่ชำรุด ไม่ซีดจางเกินไป
4. **รีสตาร์ทแอป** - ปิดและเปิด LIFF App ใหม่
5. **อัปเดต LINE** - อัปเดต LINE เป็นเวอร์ชันล่าสุด

**ถ้าใช้ Android:**
- ตั้งค่า > แอป > LINE > สิทธิ์ > กล้อง > อนุญาต

**ถ้าใช้ iOS:**
- ตั้งค่า > LINE > กล้อง > เปิด

### ⚠️ QR Code ไม่พบในระบบ

**อาการ:** สแกนสำเร็จแต่ขึ้นว่า "QR Code ไม่พบในระบบ"

**วิธีแก้:**

1. QR Code อาจสร้างมาจากระบบเก่า หรือถูกลบออกแล้ว
2. **ติดต่อผู้ดูแลฟาร์ม** เพื่อสร้าง QR Code ใหม่
3. ถ้าเป็นฟาร์มใหม่ อาจต้องรอ admin สร้างข้อมูลก่อน

### 📴 ไม่สามารถบันทึกข้อมูล Offline

**อาการ:** ไม่มีเน็ต และบันทึกข้อมูลไม่ได้

**วิธีแก้:**

1. **ตรวจสอบการเชื่อมต่อ** - ลองเปิดเว็บอื่นดูว่ามีเน็ตหรือไม่
2. **เปิดโหมดเครื่องบินแล้วปิด** - บังคับให้มือถือเชื่อมต่อใหม่
3. **ย้ายที่** - ถ้าอยู่ในพื้นที่มืดสัญญาณ ลองย้ายไปที่ที่มีสัญญาณดีกว่า
4. **ตรวจสอบว่า LIFF App เปิดโหมด Offline** - ควรมีข้อความบอกที่ด้านบน

### 🔄 Sync ไม่สำเร็จ

**อาการ:** ข้อมูลที่บันทึก Offline ไม่ส่งขึ้นระบบ

**วิธีแก้:**

1. **ตรวจสอบเน็ต** - ต้องมีเน็ตที่เสถียร
2. **เปิด LIFF App** - ให้ app sync อัตโนมัติ
3. **กด Sync ด้วยตนเอง** - แตะปุ่ม sync ในแถบสถานะ
4. **รีสตาร์ทแอป** - ปิดและเปิด LIFF App
5. **ตรวจสอบว่าข้อมูลยังอยู่** - ข้อมูลจะอยู่ใน "รายการรอ sync" เสมอ

**ถ้า sync ล้มเหลวซ้ำ ๆ:**
- ถ่ายรูปใหม่ (รูปอาจเสียหาย)
- ตรวจสอบว่ามีพื้นที่ในเครื่องพอ
- ติดต่อผู้ดูแล

### 📉 ข้อมูล Offline สูญหาย

**อาการ:** ข้อมูลที่บันทึก Offline ไม่พบ

**วิธีแก้:**

1. ข้อมูลอาจ sync ไปแล้วแต่ไม่แสดง - ตรวจสอบใน Dashboard ของ Admin
2. ถ้าล้าง cache หรือลบแอป ข้อมูล Offline จะสูญ
3. **ป้องกัน:** ควร sync ข้อมูลก่อนล้าง cache

## 3. ปัญหา Dashboard (ไม่แสดงข้อมูล)

### 📊 Dashboard ไม่แสดงข้อมูล

**อาการ:** หน้า Dashboard โหลดนาน หรือแสดงว่า "ไม่มีข้อมูล"

**วิธีแก้:**

1. **รีเฟรชหน้า** (F5 หรือ Ctrl+R)
2. **ตรวจสอบสิทธิ์** - Worker อาจไม่มีสิทธิ์ดู Dashboard เต็มรูปแบบ
3. **ตรวจสอบวันที่** - อาจกำลังดูช่วงวันที่ไม่มีข้อมูล
4. **ตรวจสอบ Server** - Server อาจปิดหรือมีปัญหา
5. **ลองเบราว์เซอร์อื่น** - Chrome, Firefox, Edge

### 🔄 กราฟหรือสถิติไม่โหลด

**อาการ:** กราฟแสดงเป็นกรอบว่าง ๆ หรือตัวเลขเป็น 0

**วิธีแก้:**

1. **รอสักครู่** - ข้อมูลอาจกำลังโหลด
2. **ตรวจสอบช่วงวันที่** - ถ้าไม่มีกิจกรรมในช่วงนั้น กราฟจะเป็น 0
3. **ตรวจสอบ Logs** - ดูว่ามี error ใน Console ของเบราว์เซอร์หรือไม่
   - กด F12 > Console > ดู error
4. **ติดต่อ super_admin** ถ้าปัญหายังอยู่

### 📝 ข้อมูลกิจกรรมไม่แสดง

**อาการ:** ไม่เห็นกิจกรรมที่บันทึกไว้

**วิธีแก้:**

1. **ตรวจสอบสิทธิ์** - ต้องเป็น owner, manager หรือ super_admin ถึงจะดูได้
2. **ตรวจสอบฟาร์ม** - ดูว่าเลือกฟาร์มถูกต้องหรือไม่
3. **ตรวจสอบช่วงวันที่** - เปลี่ยนช่วงวันที่ดู
4. **ตรวจสอบ Sync** - ถ้าบันทึก Offline ต้องรอ sync ก่อน

### ⚠️ Error 500, 502, 503

**อาการ:** หน้าเว็บแสดง Error หมายเลข

**วิธีแก้:**

1. **รีเฟรชหน้า** - อาจเป็นปัญหาชั่วคราว
2. **รอ 1-2 นาที** แล้วลองใหม่
3. **ตรวจสอบ Server** - Server อาจกำลังบำรุงรักษา
4. **ติดต่อ super_admin** พร้อมแจ้งหมายเลข Error

## 4. ปัญหา Docker

### 🐳 Container ไม่ Start

**อาการ:** Container ไม่ทำงาน หรือ Restart ตลอด

**วิธีแก้:**

```bash
# ดูสถานะ Container
docker ps -a

# ดู logs ของ Container ที่มีปัญหา
docker logs <container_name>

# ลอง restart
docker restart <container_name>
```

### 📝 ดู Logs ของ Docker Container

```bash
# ดู logs ล่าสุด
docker logs --tail 100 <container_name>

# ดู logs แบบต่อเนื่อง
docker logs -f <container_name>

# ดู logs ตามเวลา
docker logs --since "2026-03-26T10:00:00" <container_name>
```

### 💾 Docker ใช้พื้นที่ดิสก์มากเกินไป

**อาการ:** Server ช้า หรือ Docker แจ้งเตือนพื้นที่เต็ม

**วิธีแก้:**

```bash
# ดูขนาดพื้นที่ที่ Docker ใช้
docker system df

# ลบ image, container, network ที่ไม่ใช้งาน
docker system prune -a

# ลบ volume ที่ไม่ใช้งาน
docker volume prune
```

### 🔗 Port ถูกใช้งานแล้ว

**อาการ:** Error "port is already allocated"

**วิธีแก้:**

```bash
# ดูว่า port ถูกใช้งานอะไร
lsof -i :80
# หรือ
netstat -tlnp | grep :80

# เปลี่ยน port ใน docker-compose.yml
# หรือหยุด service ที่ใช้ port นั้น
```

### 🔄 Docker Compose ไม่ทำงาน

**อาการ:** docker-compose up แล้ว error

**วิธีแก้:**

```bash
# หยุด container ทั้งหมด
docker-compose down

# ลบ volume (ถ้าต้องการล้างข้อมูล)
docker-compose down -v

# สร้างใหม่
docker-compose up -d

# ถ้ายังมีปัญหา ดู logs
docker-compose logs -f
```

### 📦 Build Docker Image ล้มเหลว

**อาการ:** docker-compose build แล้ว error

**วิธีแก้:**

```bash
# ลบ cache แล้ว build ใหม่
docker-compose build --no-cache

# ตรวจสอบว่า Dockerfile ถูกต้อง
cat Dockerfile

# ตรวจสอบว่า docker เวอร์ชันล่าสุด
docker --version
docker-compose --version
```

## 5. ปัญหา Network/Server

### 🌐 Server ไม่ตอบสนอง

**อาการ:** เข้าเว็บไม่ได้ หรือเข้าได้ช้ามาก

**วิธีแก้:**

1. **ตรวจสอบว่า Server ออนไลน์หรือไม่:**
   ```bash
   # ping Server
   ping <server_ip>
   
   # ตรวจสอบว่า port 80/443 เปิดอยู่
   telnet <server_ip> 80
   ```

2. **ตรวจสอบ Docker:**
   ```bash
   docker ps
   docker-compose ps
   ```

3. **ตรวจสอบ nginx/service:**
   ```bash
   # restart nginx
   sudo systemctl restart nginx
   
   # ดู status
   sudo systemctl status nginx
   ```

### 🔒 SSL Certificate หมดอายุ

**อาการ:** เบราว์เซอร์แจ้งเตือน "Your connection is not private"

**วิธีแก้:**

```bash
# ตรวจสอบวันหมดอายุ
openssl s_client -connect your-domain.com:443 | openssl x509 -noout -dates

# ต่ออายุ certificate ( Let's Encrypt)
sudo certbot renew

# หรือสร้างใหม่
sudo certbot certonly --nginx -d your-domain.com
```

### 📧 Email ไม่ส่งได้

**อาการ:** ระบบไม่ส่งอีเมล (เช่น ลืมรหัสผ่าน)

**วิธีแก้:**

1. **ตรวจสอบ SMTP settings:**
   - ตรวจสอบว่า SMTP credentials ถูกต้อง
   - ตรวจสอบว่า SMTP server ทำงานอยู่

2. **ดู logs:**
   ```bash
   docker logs <app_container> | grep mail
   docker logs <mail_container>
   ```

3. **ตรวจสอบ firewall:**
   ```bash
   # ดูว่า port 587 หรือ 465 เปิดอยู่
   sudo ufw status
   ```

### 💾 Server ช้ามาก

**อาการ:** เว็บโหลดช้า 5-10 วินาทีขึ้นไป

**วิธีแก้:**

1. **ตรวจสอบทรัพยากร Server:**
   ```bash
   # CPU, Memory
   top
   htop
   
   # Disk I/O
   iostat -x 1 5
   
   # Network
   iftop
   ```

2. **ตรวจสอบ Database:**
   ```bash
   # เข้า container
   docker exec -it <db_container> psql -U postgres
   
   # ดู connection ที่ค้าง
   SELECT * FROM pg_stat_activity WHERE state = 'active';
   ```

3. **ตรวจสอบ Docker:**
   ```bash
   docker stats
   ```

## 6. คำถามที่พบบ่อย (FAQ)

### ทั่วไป

**Q: ระบบรองรับกี่ภาษา?**
A: ปัจจุบันรองรับ 2 ภาษา: ไทย และ อังกฤษ

**Q: สามารถใช้งานบนแท็บเล็ตได้หรือไม่?**
A: ได้ Admin Panel รองรับทั้ง Desktop, Tablet และ Mobile

**Q: ข้อมูลถูกสำรองอย่างไร?**
A: ข้อมูลถูกสำรองอัตโนมัติทุกวัน และสามารถ export เป็นไฟล์ได้

### LIFF App

**Q: LIFF App ต้องต่อเน็ตตลอดเวลาหรือไม่?**
A: ไม่จำเป็น สามารถบันทึกข้อมูล Offline ได้ แต่ต้อง sync เมื่อกลับมาออนไลน์

**Q: ถ้าโทรศัพท์หาย ข้อมูลที่บันทึก Offline จะหายไหม?**
A: ข้อมูลอยู่ในเครื่องโทรศัพท์ ถ้าเครื่องหายข้อมูลจะหายด้วย แนะนำ sync ข้อมูลก่อนทุกครั้ง

**Q: สามารถแนบรูปถ่ายได้กี่รูปต่อครั้ง?**
A: สามารถแนบได้สูงสุด 5 รูปต่อการบันทึกกิจกรรม

**Q: รูปถ่ายถูกบีบอัดหรือไม่?**
A: ใช่ รูปจะถูกบีบอัดเป็นขนาดเล็กลงเมื่อบันทึก Offline เพื่อประหยัดพื้นที่

### Admin Panel

**Q: Super Admin มีได้กี่คน?**
A: สามารถมีได้หลายคน แต่แนะนำ 1-2 คนเพื่อความปลอดภัย

**Q: สามารถเปลี่ยนเจ้าของฟาร์มได้หรือไม่?**
A: ได้ โดย super_admin สามารถแก้ไขข้อมูล Owner ของฟาร์มได้

**Q: ถ้าต้องการเพิ่มพืชใหม่ที่ไม่มีในรายการ?**
A: Owner หรือ Manager สามารถเพิ่มพืชใหม่ได้เลยจากหน้าจัดการพืช

**Q: QR Code สามารถพิมพ์ซ้ำได้หรือไม่?**
A: ได้ สามารถเข้าเมนู QR Code Management แล้วดาวน์โหลดหรือพิมพ์ใหม่ได้

### ปัญหาทางเทคนิค

**Q: ถ้า Server ล่ม ข้อมูลจะหายไหม?**
A: ไม่หาย ข้อมูลเก็บใน Database Server แยกต่างหาก และมีการสำรองข้อมูลเป็นประจำ

**Q: Docker ควรอัปเดตบ่อยแค่ไหน?**
A: แนะนำอัปเดตเมื่อมี security patch หรือ version ใหม่ที่มีฟีเจอร์สำคัญ

**Q: สามารถเพิ่ม Storage ของ Server ได้หรือไม่?**
A: ได้ แต่ต้องทำผ่าน Cloud Provider หรือเพิ่ม Harddisk แล้ว mount ใหม่

**Q: ถ้าลืมรหัสผ่าน super_admin?**
A: ต้องติดต่อผู้ดูแล Server โดยตรงเพื่อ reset จากฐานข้อมูล หรือใช้วิธี recovery ผ่าน SSH
